Arch is a high-performance, archetype-based Entity Component System (ECS) for C# and .NET, designed for game development and data-oriented programming. It stores entities in archetypes backed by 16KB cache-friendly chunks, giving you iteration speeds that rival C++ and Rust ECS libraries — all with a clean, minimal API.

## Why Arch?

Arch is built for developers who need maximum performance without sacrificing ergonomics. It uses a classic archetype layout — grouping entities with identical component sets into contiguous memory chunks — ensuring tight cache usage even for worlds with millions of entities.

<CardGroup cols={3}>
  <Card title="Cache-Efficient" icon="bolt">

  </Card>
  <Card title="Multithreaded" icon="microchip">

## Get started in three steps

<Steps>
  <Step title="Install the NuGet package">
    Add Arch to your project with the .NET CLI:
    ```bash
    dotnet add package Arch --version 2.1.0-beta
    ```
  </Step>
  <Step title="Create a world and entities">
    A `World` is the container for all your entities. Create one and spawn your first entity with components:
    ```csharp
    using Arch.Core;

    public record struct Position(float X, float Y);
    public record struct Velocity(float Dx, float Dy);

    using var world = World.Create();
    var entity = world.Create(new Position(0, 0), new Velocity(1, 1));
    ```
  </Step>
  <Step title="Query and update entities">
    Use `QueryDescription` to filter entities by their components, then iterate with a lambda:
    ```csharp
    var query = new QueryDescription().WithAll<Position, Velocity>();
    world.Query(in query, (ref Position pos, ref Velocity vel) => {
        pos.X += vel.Dx;
        pos.Y += vel.Dy;
    });
    ```
  </Step>
</Steps>
